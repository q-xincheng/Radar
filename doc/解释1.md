这份题目要求你开发一个名为“行研雷达”的动态监控智能体，其核心逻辑在于**“增量更新”**与**“冲突解决”**。

要完成这个任务，你可以按照以下四个步骤进行架构设计和开发：

---

## 1. 架构设计：云原生数据流水线

利用图中的“云原生”参考建议，你可以构建一个无服务器（Serverless）的工作流：

- **触发层 (Cron Triggers)：** 使用阿里云 FC 或 AWS Lambda 的定时触发器（如每 6 小时一次）。
    
- **采集层 (Scraper Agent)：** 编写一个爬虫脚本，根据输入的“行业关键词”在全网（如新闻 API、雪球、巨潮资讯等）抓取最新资讯。
    
- **存储层 (Object Storage)：** 将每次抓取的结果存入 S3 或 OSS，文件名建议带上时间戳（如 `report_20260119.json`），以便后续做对比。
    

## 2. 核心功能实现逻辑

### **功能一：增量对比 (Incremental Analysis)**

这是本题的核心难点。你不能只是简单的文本对比，而需要利用 **LLM（大语言模型）的提取能力**：

1. **数据提取：** 将“旧结论”和“新资讯”同时输入 LLM。
    
2. **Prompt 设计：** > “请对比以下两段信息。旧结论中提到增长率为 5%，新资讯提到最新预测为 2%。请识别出这种数值变动，并以 JSON 格式输出：`{field: '增长率', old: '5%', new: '2%', status: 'decreased'}`。”
    
3. **结果持久化：** 仅记录有变动的字段。
    

### **功能二：冲突仲裁 (Conflict Resolution)**

当多个信息源对同一个指标有不同描述时，引入**权重系统**：

- **优先级逻辑：** 官方公告 (Weight: 1.0) > 权威媒体 (Weight: 0.7) > 市场传闻 (Weight: 0.3)。
    
- **代码实现：** 在智能体的 Decision 模块中硬编码一套来源标签（Tag）。如果新旧数据冲突，自动选择权重最高的来源作为“最终结论”，并将低权重的设为“待核实”。
    

## 3. 推荐技术栈 (最小可行性方案)

|**模块**|**推荐工具**|
|---|---|
|**编程语言**|Python (处理文本和 LLM 调用最方便)|
|**大模型框架**|LangChain 或 LangGraph (处理状态流转)|
|**云服务**|阿里云 FC (Function Compute) + OSS|
|**LLM API**|GPT-4o 或 Qwen-Max (适合处理中文行研数据)|

## 4. 实施建议与考核提点

为了拿高分，你在演示或提交时应重点突出以下几点：

1. **数据可视化：** 不要只给后台日志。做一个简单的 Markdown 报告或前端页面，用**红色/绿色高亮**显示数值的升降（即考核指标中的“高亮显示”）。
    
2. **鲁棒性：** 说明你如何处理爬虫失效的情况（例如：如果某次巡检抓不到数据，系统应报警而不是覆盖旧数据）。
    
3. **Prompt 调优：** 展示你如何通过 Prompt 让 AI 准确识别“5% 调至 2%”这种细微的语义变化。
